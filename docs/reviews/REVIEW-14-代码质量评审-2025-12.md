### 发现的代码问题

#### 严重 (必须修复)
1. JWT 密钥存在默认回退值，生产配置缺失时会以弱密钥启动
   - 文件：`c:\\workspace\\bobcrm\\src\\BobCrm.Api\\Program.cs:89`
   - 问题代码：
   ```csharp
   var key = Encoding.UTF8.GetBytes(builder.Configuration["Jwt:Key"] ?? "dev-secret-change-in-prod-1234567890");
   ```
   - 建议修复：
   ```csharp
   var jwtKey = builder.Configuration["Jwt:Key"];
   if (string.IsNullOrWhiteSpace(jwtKey)) throw new InvalidOperationException("Jwt:Key is required");
   var key = Encoding.UTF8.GetBytes(jwtKey);
   ```

2. CORS 默认 AllowAnyOrigin/AllowAnyHeader/AllowAnyMethod，若误用于生产将扩大攻击面
   - 文件：`c:\\workspace\\bobcrm\\src\\BobCrm.Api\\Program.cs:272`
   - 问题代码：
   ```csharp
   policy.AllowAnyOrigin().AllowAnyHeader().AllowAnyMethod()
   ```
   - 建议修复：根据环境切换白名单（AllowedOrigins），并考虑开启凭证/方法限制

#### 重要 (建议修复)
1. API 默认成功消息硬编码中文，易导致多语言与客户端一致性问题
   - 文件：`c:\\workspace\\bobcrm\\src\\BobCrm.Api\\Contracts\\ApiResponseExtensions.cs:9`
   - 问题代码：
   ```csharp
   new(message ?? "操作成功");
   ```
   - 建议修复：取消默认文案（仅返回 code/key），或在上层通过 ILocalization 注入生成多语消息

2. 全局异常处理中间件/过滤器返回文案硬编码中文，且存在实现重复
   - 文件：`c:\\workspace\\bobcrm\\src\\BobCrm.Api\\Middleware\\GlobalExceptionMiddleware.cs:106`
   - 文件：`c:\\workspace\\bobcrm\\src\\BobCrm.Api\\Filters\\GlobalExceptionFilter.cs:96`
   - 问题点：生产环境返回中文固定文案 + 两套映射逻辑重复，维护成本高
   - 建议修复：统一到单一异常处理策略（推荐中间件），错误返回只给 errorCode/messageKey，前端用 i18n 渲染

3. 前端存在大量空 catch，可能吞掉关键错误（JS 互操作/状态刷新等）
   - 文件：`c:\\workspace\\bobcrm\\src\\BobCrm.App\\Services\\AuthService.cs:53`
   - 文件：`c:\\workspace\\bobcrm\\src\\BobCrm.App\\Components\\Shared\\EntitySelector.razor:178`
   - 建议修复：至少限定异常类型（例如 JSException）并记录到 console/ILogger；若确需忽略，写明原因

4. Blazor 事件处理使用 async void，异常不可控且难测试
   - 文件：`c:\\workspace\\bobcrm\\src\\BobCrm.App\\Components\\Shared\\EntitySelector.razor:184`
   - 文件：`c:\\workspace\\bobcrm\\src\\BobCrm.App\\Components\\Pages\\Profile.razor:297`
   - 文件：`c:\\workspace\\bobcrm\\src\\BobCrm.App\\Components\\Pages\\Templates.razor:368`
   - 建议修复：改为 `async Task` 并在调用处 `await`；必要时封装 `SafeInvokeAsync(Func<Task>)`

5. 可空性告警：ThrowDomain 之后编译器仍认为对象可为 null，导致 CS8602；模板运行时存在潜在空引用/参数为空风险
   - 文件：`c:\\workspace\\bobcrm\\src\\BobCrm.Api\\Base\\Aggregates\\EntityDefinitionAggregate.cs:70`
   - 文件：`c:\\workspace\\bobcrm\\src\\BobCrm.Api\\Services\\TemplateRuntimeService.cs:81`
   - 问题代码：
   ```csharp
   if (subEntity == null) { ThrowDomain("ERR_SUBENTITY_NOT_FOUND", subEntityId); }
   subEntity.DisplayName = displayName;
   ```
   - 建议修复：为 ThrowDomain 添加 `[DoesNotReturn]` 或内联 `throw`；TemplateRuntimeService 使用 normalized entityType 并确保 bindingToUse.Template 非空

6. 过时 API 大量残留（CS0618：FunctionNode.TemplateId/TemplateBindingId 等），迁移成本持续累积
   - 文件：`c:\\workspace\\bobcrm\\src\\BobCrm.Api\\Services\\AccessService.cs:225`

8. DTO 非空字段可能接收 null（CS8601），需补齐默认值或调整 DTO 可空性
