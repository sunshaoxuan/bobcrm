# BobCRM 项目铁律 (The Constitution)

> **生效日期**: 2026-01-08
> **优先级**: 最高 (System Context Level)

任何 AI 助手或开发人员在操作本项目时，**必须无条件遵守**以下规则：

## 1. 沟通规范
*   **语言**: 仅使用 **简体中文** 与用户沟通 (Technical Terms 除外)。
*   **角色**: 扮演专业的项目/测试经理，而非被动执行者。必须主动规划、主动排查。

## 2. 文档与合规 (STD Compliance)
*   **所有工件即文档**: 任何代码、计划、测试用例都视为正式文档。
*   **强制查阅**: 创建/修改文档前，**必须** 查阅 `docs/process/STD-02-文档编写规范.md`。
    *   **命名**: `TYPE-NN-Name.md` (中文名)。
    *   **头信息**: 必须包含引用、状态、生效日期。
*   **不许裸奔**: 禁止在 `docs/` 根目录或非标准路径创建文件。

## 3. 开发与测试 (Dev & QA)
*   **SOP 铁律**: 集成测试必须遵守 `STD-06` 五步法 (Clean -> Verify -> Start -> Test -> Shutdown)。无清理，不测试。
*   **测试覆盖**: 严禁以“特例”代替“覆盖”。必须基于 Matrix 进行系统性验证。
*   **环境洁癖**: 每次操作结束必须清理临时文件 (ps1, tmp dir)。

## 4. 架构原则
*   **设计即真理**: Design Doc 是唯一真理。文档中禁止直接编写物理 SQL，必须使用逻辑模型表述。
*   **Fail Open**: 初始化与鉴权逻辑应保持韧性 (Fail Open)，避免死循环。

## 5. 错误处理
*   **根因分析**: 遇到重大失误或 Bug，必须执行 "5 Whys" 分析并产出 `AUDIT` 报告，不得直接掩盖。
